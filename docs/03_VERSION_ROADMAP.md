# 03 版本路线

## 文档目的

记录项目从骨架到可用订阅池的版本推进计划。

## 当前版本范围

- v0.1.0：项目骨架和配置文件
- v0.2.0：GitHub 采集与限速
- v0.3.0：节点解析与节点池
- v0.3.1：节点池清理与 UI 小修
- v0.4.0：Xray-core 可用性检测
- v0.4.2：Xray 节点参数兼容增强
- v0.4.3：VLESS Reality 检测链路修复
- v0.4.4：VLESS Reality 临时 Xray 配置细节修复
- v0.4.5：节点状态手动校正
- v0.5.0：安全订阅链接基础版（当前开发版本）
- v0.5.1：自动订阅刷新
- v0.6.0：二维码和订阅有效期
- v0.7.0：统计面板完善
- v0.8.0：sing-box / Mihomo 内核管理
- v0.9.0：Telegram Bot 只读订阅接口

## 后续待补充内容

补充每个版本的验收标准、数据结构变更和回滚方案。

## v0.6.0 更新

- v0.5.0：安全订阅链接基础版
- v0.6.0：自动刷新订阅内容基础版（当前版本）
- v0.6.1：二维码和订阅有效期

## v0.6.1 更新

- v0.6.1：订阅管理页隐藏安全订阅链接明文（当前版本）
- v0.6.2：二维码和订阅有效期

v0.6.1 只调整前端展示和版本号，不修改订阅生成、自动刷新或 token 生成规则。

## v0.6.2 更新

- v0.6.2：修复“复制安全订阅链接”按钮在 HTTP 环境下无反馈的问题（当前版本）
- v0.6.3：二维码和订阅有效期

v0.6.2 只调整前端复制兼容逻辑和版本号，不修改后端订阅生成、自动刷新或 token 生成规则。
