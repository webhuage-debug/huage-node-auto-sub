# 03 版本路线

## 文档目的

记录项目从骨架到可用订阅池的版本推进计划。

- v0.2.0：GitHub 采集与限速
- v0.3.0：节点解析与节点池
- v0.3.1：节点池清理与 UI 小修
- v0.4.0：Xray-core 可用性检测
- v0.4.2：Xray 节点参数兼容增强
- v0.4.3：VLESS Reality 检测链路修复
- v0.4.4：VLESS Reality 临时 Xray 配置细节修复
- v0.4.5：节点状态手动校正
- v0.5.0：安全订阅链接基础版（当前开发版本）
- v0.5.1：自动订阅刷新
- v0.6.0：二维码和订阅有效期
- v0.7.0：订阅有效期基础版
- v0.7.1：订阅二维码展示基础版
- v0.8.0：公开领取页和口令验证基础版

## 后续待补充内容

补充每个版本的验收标准、数据结构变更和回滚方案。

## v0.6.0 更新

- v0.5.0：安全订阅链接基础版
- v0.6.0：自动刷新订阅内容基础版（当前版本）
- v0.6.1：二维码和订阅有效期

v0.6.0 只负责按配置刷新订阅缓存，不做二维码、领取页、Telegram Bot、自动采集或自动检测。

## v0.6.1 更新

- v0.6.1：订阅管理页隐藏安全订阅链接明文（当前版本）
- v0.6.2：二维码和订阅有效期

v0.6.1 只调整前端展示和版本号，不修改订阅生成、自动刷新或 token 生成规则。

## v0.6.2 更新

- v0.6.2：修复“复制安全订阅链接”按钮在 HTTP 环境下无反馈的问题（当前版本）
- v0.6.3：二维码和订阅有效期

v0.6.2 只调整前端复制兼容逻辑和版本号，不修改后端订阅生成、自动刷新或 token 生成规则。

## v0.6.3 更新

- v0.6.3：订阅公开访问地址配置（当前版本）
- v0.6.4：重置安全订阅链接 token

v0.6.3 将后台访问地址和粉丝订阅地址分离，复制订阅链接时使用 `SUBSCRIPTION_PUBLIC_BASE_URL`，不再使用后台当前访问域名和 3000 端口。

## v0.6.4 更新

- v0.6.4：重置安全订阅链接 token（当前版本）
- v0.6.5：二维码和订阅有效期

v0.6.4 增加后台手动重置安全订阅 token 的能力，用于 token 暴露后的快速失效处理，不改变订阅生成、自动刷新或公开域名配置规则。

## v0.7.0 更新

- v0.7.0：订阅有效期基础版
- v0.7.1：订阅二维码展示基础版（当前版本）

v0.7.0 增加活动级订阅有效期，过期后公开订阅接口返回 410，不做每用户独立有效期、登录系统、支付系统或设备限制。

## v0.7.1 更新

- v0.7.1：订阅二维码展示基础版（当前版本）
- v0.7.2：领取页或二维码体验增强

v0.7.1 在订阅管理页显示二维码，二维码内容来自公开订阅地址 `publicSubscriptionBaseUrl + safeSubscriptionUrl`。页面仍不展示完整订阅链接、`/sub/{token}` 或 token。

## v0.8.0 更新

- v0.8.0：公开领取页和口令验证基础版

v0.8.0 增加 `/claim` 公开领取页和 `POST /api/claim/verify` 口令验证接口。公开页不显示后台菜单、后台信息、完整订阅链接、`/sub/{token}` 或 token。

## v0.8.1 更新

- v0.8.1：领取验证成功后返回可复制订阅链接（当前版本）
- v0.8.2：领取页公开入口策略完善

v0.8.1 让 `POST /api/claim/verify` 在口令正确时返回 `copyableSubscriptionUrl`，公开页不再依赖 `/api/subscription/status`，避免用户绕过口令直接读取订阅状态。

## v0.8.2 更新

- v0.8.2：领取口令防刷和隐藏二维码预览（当前版本）

v0.8.2 为公开领取口令验证接口增加内存级按 IP 防刷：错误次数达到阈值后进入冷却期并返回 429；口令正确后清除失败计数。同时订阅管理页隐藏二维码图片预览，只保留二维码状态和下载按钮，点击下载时临时生成二维码 PNG。

## v0.8.3 更新

- v0.8.3：发布前检查页基础版（当前版本）

v0.8.3 增加 `GET /api/publish-check/status`，用于正式发视频前检查订阅、有效期、节点数量、公开域名、领取口令、公开领取页和后台 API 暴露状态。系统设置页显示检查摘要和固定发布提醒，不显示 token、完整订阅链接、真实口令或 raw/base64 内容。

## v0.8.4 更新

- v0.8.4：发布前检查结果优化（当前版本）

v0.8.4 将 `CLAIM_TOO_MANY_ATTEMPTS` 识别为公开验证接口可达但已进入防刷冷却期，状态为 warning；`INVALID_CLAIM_CODE` 继续视为接口可达并通过。后台 API 暴露检查仍保持严格，公开域名下后台 API 非 404 即为 fail。

## v0.8.5 更新

- v0.8.5：发布前一键准备操作区（当前版本）

v0.8.5 增加 `POST /api/publish-check/prepare`，用于后台执行发布前准备：重置安全订阅 token、续期订阅有效期，并返回脱敏结果。系统设置页发布前检查区域新增按钮，执行成功后自动刷新检查结果。
# v0.9.0 发布候选版更新

- v0.9.0：发布候选版 RC（当前版本）
- 本版本不新增业务功能，只整理 README 发布说明、部署说明、正式发布前检查清单、回滚说明、最终验收脚本和 Codex 固定工作规则。
- 发布要求继续保持：不显示 token、不显示完整订阅链接、不显示真实口令、不显示 raw/base64 内容，不修改 Caddy/Docker，不提交 `data/` 或 `cores/`。
# v1.0.1 热修复

- v1.0.1：修复 Xray-core 内核状态识别和真实检测可用性验收。
- 后端通过执行 `/app/cores/xray/xray version` 判断 Xray-core 是否存在、可执行并解析版本号。
- 内核管理页显示 Xray-core 已安装、版本号、可执行路径和失败原因。
- 本版本不新增业务功能，不修改订阅、领取、口令、防刷、节点池或 Caddy/Docker 配置。

# v1.0.2 热修复

- v1.0.2：修复 Xray 检测流程误判问题。
- 软件检测流程对齐手动成功流程：临时配置、`xray run -config`、等待 SOCKS 端口、`curl --socks5-hostname` 检测。
- `http_code=204` 或 `http_code=200` 判定为 `available`。
- 增加安全 debug 字段用于定位检测阶段，不返回 raw 节点、token、完整订阅链接或完整 publicKey。
- 本版本不新增业务功能，不修改订阅、领取、口令、防刷、节点池或 Caddy/Docker 配置。

# v1.0.0 稳定版更新

- v1.0.0：稳定版封版（当前稳定版）
- 本版本不新增业务功能，只更新版本号、README、稳定版发布说明、运行数据备份脚本和最终验收清单。
- 后续开发必须另开新版本，不能在 v1.0.0 稳定版上混入新业务。
