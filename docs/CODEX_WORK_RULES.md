# Codex 固定工作规则

## 文档目的

整理本项目后续 Codex 开发和发布时必须遵守的固定规则，减少重复提示词。

## 当前版本范围

适用于 `huage-node-auto-sub` 后续所有小版本开发、修复、文档整理和发布候选版收口。

## 固定规则

1. 不提交 `data/`、`cores/`、`.env`。
2. 不提交数据库、日志、节点数据、内核二进制文件、token、密码、密钥。
3. 不暴露 token、raw 节点、完整订阅链接、真实口令。
4. 不随意修改 `Dockerfile`、`docker-compose.yml`、Caddy 或系统反向代理配置。
5. 每次只做一个明确小版本，不把后续版本功能提前混入。
6. 每次任务必须写清允许修改文件和禁止修改文件。
7. 每次必须尝试执行 `npm run build`，如果无法执行，要明确说明原因。
8. 每次完成后必须 commit。
9. 每次不要 push，等待人工确认后再推送。
10. 输出必须包含修改文件、测试结果、风险和下一步建议。
11. 所有公开入口只能开放必要路径，不能开放全部 `/api/*`。
12. 公开域名只应放行 `/claim`、`/assets/*`、`/api/claim/verify` 和 `/sub/*`。
13. 后台 API、节点池、检测接口、订阅状态接口不能暴露给粉丝访问。
14. 正式发布前必须最后重置一次安全订阅 token，并确认视频口令是本期口令。

## 后续待补充内容

如果后续引入多活动、统计、Telegram Bot 或数据库，再补充对应固定规则。
