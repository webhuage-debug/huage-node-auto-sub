# 14 正式发布前检查清单

## 文档目的

把正式发视频前必须确认的事项固定下来，避免遗漏安全订阅、口令、公开入口和后台隔离检查。

## 当前版本范围

本文适用于 `v0.9.0` 发布候选版。

## 发布前检查

- 版本检查：后台 `/api/status` 返回 `v0.9.0`。
- 订阅检查：安全订阅已生成。
- 有效期检查：订阅未过期，剩余有效期足够本期视频使用。
- 节点数量检查：订阅节点数量不为 0，最好达到最低保底数量。
- 口令检查：`CLAIM_ACCESS_CODE` 已改成本期视频口令，不能沿用旧视频口令。
- 领取页检查：`https://get.huage.us/claim` 可访问。
- 验证接口检查：`/api/claim/verify` 可用，错误口令返回错误或防刷提示。
- 后台隔离检查：公开域名下 `/api/status` 返回 404。
- 订阅状态隔离检查：公开域名下 `/api/subscription/status` 返回 404。
- 节点池隔离检查：公开域名下 `/api/node-pool/status` 返回 404。
- 检测接口隔离检查：公开域名下 `/api/detection/xray/status` 返回 404。
- 最后安全动作：正式发布前最后重置一次安全订阅 token。
- 有效期动作：正式发布前续期订阅有效期。
- 二维码动作：重置 token 后重新下载二维码。

## 不允许出现

- 不截图 token。
- 不公开完整订阅链接。
- 不公开后台地址。
- 不公开 3000 端口。
- 不开放全部 `/api/*`。
- 不把 `.env`、`data/`、`cores/` 提交到 Git。

## 推荐操作顺序

1. 修改 VPS 环境变量中的 `CLAIM_ACCESS_CODE`。
2. 在后台系统设置页执行发布前准备。
3. 刷新发布前检查。
4. 下载新的二维码。
5. 打开 `/claim` 测试错误口令。
6. 打开 `/claim` 测试正确口令并复制订阅。
7. 确认公开域名下后台 API 全部被 404 挡住。
